핵심 요구사항 분석

  • 상세 기획 — 요구사항 정의, 기능 정의, 화면 설계(IA) 수행
  • 회원가입 및 로그인 — 서비스 이용을 위한 기본 인증 플로우 디자인
  • 프로필 관리 — 사용자 프로필 정보 등록/수정 화면 디자인
  • 사용자 탐색 — 카드 스와이프 기반 프로필 탐색 UI (틴더 스타일 벤치마킹)
  • 매칭 기능 — 관심 표현 및 상호 매칭 성사/결과 확인 디자인
  • 커뮤니케이션 — 매칭 후 1:1 메시지/채팅 화면 디자인
  • 산출물 — Figma 디자인 파일, 주요 화면 UI, 컴포넌트 및 스타일 가이드

제안 포인트

유사한 매칭/네트워킹 앱(Connectin) 및 B2C 모바일 앱(Fortune App, Calendar Share)의 기획~디자인 수행 경험을 바탕으로 다음과 같은 차별화된 접근을 제안합니다:

  • 벤치마킹 기반 UX 설계 — 틴더(카드 스와이프), 위피(커뮤니티), 골드스푼(프리미엄 매칭), 알파스테이트(프로필 중심) 각 서비스의 핵심 UX 패턴을 분석하여 최적의 사용자 경험 도출
  • 체계적인 디자인 시스템 — 컴포넌트, 색상, 타이포그래피, 아이콘 등을 체계적으로 정리한 디자인 시스템 구축으로 향후 개발 효율성 극대화
  • 크로스플랫폼 일관성 — Android/iOS 양 플랫폼의 디자인 가이드라인(Material Design, HIG)을 고려한 일관된 UI 설계
  • 단계별 산출물 공유 — 기획 → 와이어프레임 → UI 디자인 → 스타일 가이드 순으로 단계별 검수 진행

PRD (Product Requirements Document)

프로젝트 요구사항을 정의하고 기획/디자인 범위를 명확히 합니다.

1. 프로젝트 배경 및 목적

모바일 기반의 소개팅/매칭 서비스를 준비하고 있으며, 사용자가 자신의 프로필을 등록하고 서로 관심 있는 상대를 발견하여 연결될 수 있는 서비스입니다. 직관적이고 사용성이 좋은 UI/UX를 목표로 하며, 틴더/위피/골드스푼/알파스테이트를 벤치마킹 대상으로 합니다.

본 프로젝트의 범위는 상세 기획(IA)과 Figma UI/UX 디자인이며, 개발은 별도 진행 예정입니다.

2. 사용자 정의

사용자 유형설명주요 기능
일반 사용자소개팅/매칭 서비스를 이용하는 개인 사용자회원가입, 프로필 등록, 사용자 탐색, 매칭, 채팅
관리자서비스 운영 및 관리 담당자사용자 관리, 신고 관리, 서비스 설정 (범위 협의 필요)

3. 기능 요구사항

3.1 회원가입 및 로그인

  • [필수] 소셜 로그인 (카카오, Google, Apple)
  • [필수] 본인 인증 (휴대폰 번호)
  • [필수] 이용 약관 및 개인정보 처리방침 동의
  • [필수] 온보딩 플로우 (서비스 소개)

3.2 프로필 관리

  • [필수] 프로필 사진 등록 (최소 1장, 최대 6장)
  • [필수] 기본 정보 입력 (닉네임, 나이, 성별, 키, 직업 등)
  • [필수] 자기소개 작성
  • [필수] 관심사/취미 태그 선택
  • [선택] 이상형 조건 설정 (나이, 거리, 키 등)

3.3 사용자 탐색

  • [필수] 카드형 프로필 탐색 (좌/우 스와이프)
  • [필수] 프로필 상세 보기 (탭하여 확대)
  • [필수] 필터 설정 (거리, 나이, 성별 등)
  • [선택] 프로필 신고/차단

3.4 매칭 기능

  • [필수] 좋아요/패스 관심 표현 (스와이프 또는 버튼)
  • [필수] 상호 관심 확인 시 매칭 성사 알림
  • [필수] 매칭 목록 관리 (매칭된 사용자 리스트)
  • [선택] 슈퍼라이크 기능

3.5 커뮤니케이션

  • [필수] 1:1 텍스트 채팅
  • [필수] 채팅 목록 (최근 대화 순)
  • [필수] 읽음 확인 표시
  • [선택] 이미지 전송
  • [선택] 채팅방 나가기/삭제

4. 비기능 요구사항

항목요구사항
접근성최소 터치 영역 44x44dp, 충분한 색상 대비(WCAG 2.1 AA)

성능 고려이미지 최적화 가이드, 로딩 상태 디자인 포함

5. 기술 제약사항

  • Figma 기반 작업 (다른 디자인 도구 사용 불가)
  • Android + iOS 양 플랫폼을 고려한 디자인 (Material Design 3, Human Interface Guidelines 참고)
  • 벤치마킹 대상: 틴더, 위피, 골드스푼, 알파스테이트
  • 직관적이고 사용하기 쉬운 모바일 앱 디자인 지향

6. 범위 외 사항 (Out of Scope)

  • 프론트엔드/백엔드 개발
  • 서버 구축 및 배포
  • 로고/브랜드 아이덴티티 디자인 (별도 협의)
  • 앱 스토어 등록용 스크린샷/배너
  • 관리자 페이지 디자인 (별도 협의)

Connectin — 디지털 명함 & 네트워킹 플랫폼

B2B 네트워킹 · 2025.05–2025.08 (3개월)

프로젝트 개요

BLE 근거리 탐색, OCR 명함 스캐닝, E2E 암호화 메시징을 결합한 디지털 명함 & 프로페셔널 네트워킹 플랫폼. Flutter 크로스플랫폼(iOS, Android, Web) 동시 지원.

핵심 기능

  • 프로필 기반 사용자 탐색 (BLE + GPS 근거리 발견)
  • 매칭/네트워킹 연결 및 관심 표현
  • 실시간 1:1 메시징 (Signal Protocol E2E 암호화)
  • 소셜 피드 & 게시물/댓글/리액션
  • 41개 모바일 기능 모듈, 63개 API 엔드포인트

본 프로젝트와의 유사점

프로필 기반 매칭/탐색, 관심 표현, 1:1 채팅이라는 핵심 UX 플로우가 소개팅 앱과 동일합니다. 사용자 프로필 등록 → 탐색 → 관심 표현 → 매칭 → 채팅이라는 전체 여정을 설계·구현한 경험이 있으며, 이를 소개팅 앱의 맥락에 맞게 재설계할 수 있습니다.

Flutter Dart Next.js Express PostgreSQL Figma
